# D&D Sheet

Русскоязычный веб-лист персонажа для D&D 5e с синхронизацией через Firebase, PWA-режимом, бестиарием кампании, хронологией и расширением для Owlbear Rodeo.

Проект сделан как лёгкий набор статических файлов: его можно открыть локально, положить на GitHub Pages или любой другой статический хостинг. Основной лист работает в браузере, а общая комната и данные кампании синхронизируются через Firebase Realtime Database.

## Что умеет

- Лист персонажа D&D 5e на русском языке.
- Создание персонажа 1 уровня через пошаговый мастер.
- Повышение уровня через мастер с ХП, ASI, классовыми и расовыми особенностями.
- Поддержка классов, расовых пресетов, подклассов, атак, брони, КД, ресурсов и заклинаний.
- Список заклинаний с фильтрами, изучением, подготовкой и отдельным счётчиком дополнительных заклинаний.
- Синхронизация листов игроков в комнате Firebase.
- Режим мастера: первый вошедший в комнату становится ГМом и видит листы игроков.
- Бестиарий кампании с NPC, монстрами, фракциями, тегами, отношениями, изображениями и настройками видимости для игроков.
- Хронология кампании с видимыми и скрытыми событиями, картинками и перетаскиванием карточек для ГМа.
- PWA: сайт можно установить на телефон или компьютер как приложение.
- Owlbear Rodeo-плагин: привязка токенов к листам, отображение ХП, временных ХП, КД, быстрых атак, умений и заклинаний.

## Быстрый старт

Откройте `index.html` в браузере. Для локального просмотра можно просто открыть файл, но для PWA, сервис-воркера и более близкого к GitHub Pages поведения лучше запустить простой сервер:

```bash
python3 -m http.server 8000
```

После этого откройте:

```text
http://localhost:8000/
```

## Размещение на GitHub Pages

1. Положите файлы проекта в репозиторий.
2. Включите GitHub Pages в настройках репозитория.
3. Если репозиторий называется `dndsheet`, основной адрес будет похож на:

```text
https://greenred451.github.io/dndsheet/
```

4. Для Owlbear Rodeo manifest расширения будет:

```text
https://greenred451.github.io/dndsheet/obr-sheet-link/manifest.json
```

Если имя репозитория или домен отличаются, проверьте абсолютные пути в `obr-sheet-link/manifest.json` и JS-файлах расширения. Сейчас они рассчитаны на путь `/dndsheet/...`.

## Firebase

Firebase используется для комнат, синхронизации листов игроков, бестиария и хронологии.

Конфигурация Firebase сейчас встроена в:

- `index.html`
- `bestiary.html`
- `timeline.html`
- `obr-sheet-link/background.js`
- `obr-sheet-link/main.js`
- `obr-sheet-link/context.js`
- `obr-sheet-link/spells.js`
- `obr-sheet-link/abilities.js`

Для собственного проекта нужно создать Firebase Realtime Database и заменить `FB_CONFIG` в этих файлах.

Минимальная структура данных:

```text
rooms/
  roomName/
    gm
    players/
      playerName/
    bestiary/
    timeline/
```

Пример простых правил для личной кампании без авторизации:

```json
{
  "rules": {
    "rooms": {
      "$room": {
        ".read": true,
        "gm": {
          ".write": true
        },
        "players": {
          "$player": {
            ".write": true
          }
        },
        "bestiary": {
          ".write": true
        },
        "timeline": {
          ".write": true
        }
      }
    }
  }
}
```

Важно: эти правила подходят для личной игры с друзьями, но не являются защищённой публичной моделью доступа. Если проект будет использоваться шире, лучше добавить Firebase Authentication и проверку роли ГМа на уровне правил.

## Основной лист

Откройте `index.html`.

Основные сценарии:

- `Создать персонажа` — пошаговый мастер создания героя 1 уровня.
- `Повысить уровень` — мастер повышения уровня.
- `Бестиарий` — открыть отдельную вкладку бестиария.
- `Хронология` — открыть отдельную вкладку хронологии.
- `💾` — сохранить лист в JSON.
- `📂` — загрузить лист из JSON.

Чтобы подключиться к комнате:

1. Введите название комнаты.
2. Введите имя игрока.
3. Нажмите `Войти`.

Первый вошедший в комнату становится мастером и видит обзор листов игроков.

## Бестиарий

Файл: `bestiary.html`.

Бестиарий хранится в Firebase внутри выбранной комнаты. ГМ может создавать:

- NPC;
- монстров;
- фракции;
- записи персонажей игроков, синхронизированные из комнаты.

У записей есть:

- боевая вкладка;
- мирная вкладка;
- инвентарь;
- отношения;
- теги;
- изображение по URL;
- настройки видимости для игроков.

Игроки видят только те секции, которые ГМ отметил как открытые.

## Хронология

Файл: `timeline.html`.

Хронология хранит события кампании вертикальной лентой. ГМ может:

- создавать события выше или ниже выбранного;
- перетаскивать события мышью;
- прикреплять изображения;
- скрывать дату, заголовок или подробности события от игроков.

Если событие скрыто от игроков, оно отображается как `???`.

## PWA

Проект содержит:

- `manifest.json`
- `service-worker.js`
- иконки `icon-192.png` и `icon-512.png`

После размещения на HTTPS-хостинге, например GitHub Pages, сайт можно установить как приложение через браузер.

Если после обновления файлы выглядят старыми, причина почти всегда в PWA-кэше. Увеличьте `CACHE_NAME` в `service-worker.js`, например:

```js
const CACHE_NAME = 'dnd-sheet-v81';
```

## Owlbear Rodeo-плагин

Папка: `obr-sheet-link/`.

Расширение связывает токены Owlbear Rodeo с персонажами из Firebase-комнаты листка.

Manifest для текущего GitHub Pages-пути:

```text
https://greenred451.github.io/dndsheet/obr-sheet-link/manifest.json
```

Как пользоваться:

1. Откройте комнату Owlbear Rodeo.
2. Добавьте расширение по manifest URL.
3. В расширении введите название Firebase-комнаты из листка.
4. Выберите токен на сцене.
5. Выберите персонажа.
6. Нажмите `Привязать`.

После привязки токен получает локальный оверлей:

- текущие и максимальные ХП;
- временные ХП;
- КД;
- быстрый список атак;
- окно заклинаний;
- окно умений.

Сейчас основной поток синхронизации такой:

```text
лист персонажа -> Firebase -> Owlbear Rodeo overlay
```

То есть игрок меняет ХП в листке, а токен обновляет отображение в Owlbear. Обратная запись ХП из Owlbear в листок возможна, но требует аккуратной транзакционной записи в Firebase, чтобы не затирать данные открытым листом.

## Проверка синтаксиса

В проекте много inline-JavaScript внутри HTML. Быстрая проверка:

```bash
node --check service-worker.js
```

Для проверки inline-скриптов можно извлечь их из HTML и прогнать через `node --check`. Вручную это не обязательно для обычного использования, но полезно после крупных правок.

## Ограничения

- Проект ориентирован на личные кампании, а не на публичный сервис.
- Firebase-правила без авторизации удобны, но не защищают данные от посторонних, если они знают название комнаты и адрес базы.
- Черты персонажей пока сознательно не автоматизированы.
- Некоторые сложные выборы классов и подклассов описаны текстом и могут требовать ручной записи.
- Списки и описания заклинаний адаптированы для удобства игроков и не являются заменой официальных книг правил.

## Для разработки

Проект не требует сборки. Большая часть логики находится прямо в HTML/JS-файлах.

Рекомендуемый локальный запуск:

```bash
python3 -m http.server 8000
```

Перед публикацией после изменений:

1. Проверьте синтаксис изменённых JS-файлов.
2. Если менялись PWA-файлы или основной интерфейс, увеличьте `CACHE_NAME` в `service-worker.js`.
3. Если менялся Owlbear-плагин, увеличьте версию в `obr-sheet-link/manifest.json` и query-параметры `?v=...`, чтобы Owlbear подтянул свежие файлы.

## Статус

Проект уже пригоден для игры: можно вести листы персонажей, синхронизировать комнату, смотреть листы игроков как ГМ, пользоваться бестиарием, хронологией и отображением персонажей в Owlbear Rodeo.

Следующие крупные направления развития:

- обратное изменение ХП из Owlbear в лист;
- более строгая автоматизация выборов классов и подклассов;
- черты;
- защищённая модель доступа через Firebase Authentication;
- более полный технический тест прогресса персонажей с 1 по 20 уровень.
